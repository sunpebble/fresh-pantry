# Fresh Pantry

Fresh Pantry is a local-first household pantry app: a native SwiftUI iOS client, Supabase-backed family sharing, and a thin Cloudflare Worker API surface.

## Layout

- `apps/ios` - Native SwiftUI app (Xcode project generated by XcodeGen).
- `apps/api` - Cloudflare Worker for health checks and invite deep links.
- `supabase` - Supabase migrations, tests, and local configuration.
- `docs` - design specs, ADRs, and implementation plans. The historical Flutter→SwiftUI migration blueprint lives under `docs/swiftui-migration`.

## Common Commands

```bash
npm run api:test
npm run supabase:status
```

## Local Development

### iOS

The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `apps/ios/project.yml`; the `.xcodeproj` itself is git-ignored.

```bash
cd apps/ios
xcodegen generate
open FreshPantry.xcodeproj
```

Secrets live in `apps/ios/FreshPantry/Support/Secrets.plist` (copy from `Secrets.example.plist`; the real file is git-ignored). In CI it is generated from repository secrets. See `apps/ios/README.md` for the full build, signing, and TestFlight setup.

### Supabase

```bash
npm run supabase:start
npm run supabase:reset
npx -y supabase@2.101.0 test db
```

### API

```bash
cd apps/api
npm install
npm test
npx wrangler deploy
```

The production Worker route is `api.fresh-pantry.kunish.eu.org`.

## Supabase Auth Redirect

Email OTP sign-in redirects back into the iOS app with:

```text
com.kunish.freshpantry://signin-callback/
```

Set the Supabase project's Auth Site URL to that deep link and keep the same URL in the redirect allow list before testing magic-link sign-in on devices.

The checked-in Auth resend interval is tuned for development login testing. Supabase's default SMTP still has a very low hourly email cap; configure a custom SMTP provider before relying on email auth outside personal testing.
